Introduction Why would you need to setup your own worker ? There is several cases where one would need to setup his own worker: Perform incremental build Build on a specific architecture Perform integration tests in a specific network How does this work ? Workers authenticate on CDS with a token and have the same permissions as the user who generated it. Bottom line, if you can access the application, your worker will too. »

Introduction This tutorial introduce the cds track <git commit> function of CDS cli. Goal: Immediate feedback cds track aims to display in your terminal the status of the pipeline building code refered by given commit hash. Push your branch, start cds track and get immediate feedback. Git track will display all pipelines related to given hash. This means triggered testing and deployment pipelines will be displayed.
