First Pipeline with CLI

Create a project The first step is to create your own Project. A project contains applications, pipelines and environments. You can use either the CLI or the UI to create your project. $ cds project add TEST "My test project" test-team OK Create a simple pipeline $ cds pipeline add TEST hello-pip Pipeline hello-pip created. Create an application $ cds application add TEST hello-world Application hello-world created. Configure your pipeline You can add a script step in the pipeline that simply echoes “Hello World ! »

Use Configuration File

You can define a pipeline in the json format but also in the yaml format. The default is the yaml format. Basic configuration If you have a pretty basic build pipeline with a single stage and a single job. You may write such a configuration file: steps: - script: echo I'm the firt step - script: echo I'm the second step This defines a pipeline of type build (it’s the default type), named Build (the default name for a build Pipeline). »
